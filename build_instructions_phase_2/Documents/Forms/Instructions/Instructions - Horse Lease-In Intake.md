HORSE LEASE-IN INTAKE INSTRUCTIONS

INTERNAL USE / APPLICATION BUILD REFERENCE

Purpose: This document explains how information collected from a client seeking a horse lease is used to create a lease assistance engagement.

WHEN TO PRESENT THIS FORM

Present when:

☐ Client wants to lease a horse

☐ Client wants assistance finding lease opportunities

☐ Client wants lease negotiation assistance

☐ Client wants to evaluate a lease prospect

PRIMARY INFORMATION COLLECTED

The form establishes:

☑ Lessee identity

☑ Rider experience

☑ Intended use

☑ Horse requirements

☑ Budget

☑ Desired lease terms

REQUIRED DATABASE FIELDS

Required:

Client Information

Riding Experience

Lease Objective

Horse Requirements

Budget

CONTRACT GENERATION INPUTS

Used for:

Lease Representation Agreement

Horse Lease Agreement

Evaluation Agreement (if applicable)

Populates:

☑ Client information

☑ Lease requirements

☑ Service scope

☑ Compensation

INTERNAL REVIEW ITEMS

Confirm:

☐ Rider ability matches requested horse

☐ Lease purpose is defined

☐ Budget expectations are reasonable

☐ Additional evaluation needs are identified

APPLICATION DISPLAY NOTES

Route intelligently:

If client has no horse identified:

→ Horse Search workflow

If horse is identified:

→ Evaluation / Lease workflow

COMMON ERRORS TO AVOID

Do not:

☐ Treat lease assistance as ownership transfer

☐ Guarantee suitability of lease horse

☐ Combine lease and purchase services without separate scope
